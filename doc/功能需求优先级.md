# 功能需求优先级排序

基于《功能需求文档》对功能点进行优先级排序。  
**约定**：位置相关服务、用户考核及题库功能优先级较低。  
**排期约定**：结束时间 4/27；P0 开发周期 1.5 个月；已排除法定假期（春节 2/15～2/23、清明 4/4～4/6）；按优先级顺序、尽量不交叉。

---

## 一、优先级说明

| 等级 | 说明 |
|------|------|
| **P0** | 最高优先级，MVP 必备 |
| **P1** | 高优先级，核心业务 |
| **P2** | 中优先级，社交与增强 |
| **P3** | 较低优先级（位置相关、用户考核与题库） |

---

## 六、汇总

| 优先级 | 说明 | 主要范围 | 起止时间 |
|--------|------|----------|----------|
| **P0** | MVP 必备（约 1.5 个月） | 基本框架搭建与开发、用户与认证、问答与回答、评论与基础互动、个人中心基础、导航与通知入口、邀请回答（仅 Twitter） | 2026/1/30～2026/3/18 |
| **P1** | 核心业务 | 搜索、悬赏与钱包、通知、内容权限与可见性、社交、问题团队与群组、问题设置、付费答案与仲裁 | 2026/3/19～2026/4/15 |
| **P2** | 社交与增强 | 个人中心扩展、频道与发现、热榜、活动系统、紧急求助 | 2026/4/16～2026/4/27 |
| **P3** | 较低优先级 | **位置相关**（不含紧急求助）：地区选择、问题/个人/导航/活动中的位置展示；**用户考核与题库**：智慧指数、影响力、题库、答题/考试及历史记录；**邀请扩展**：Facebook、Instagram | 4/27 后或二期 |
